stream流

发表于 2023-06-09 分类于 Java 本文字数： 6.4k
stream流创建与调用

创建操作

数组

通过调用Arrays.stream(T[])取得数组的stream流

单列集合

单列集合直接通过集合中所属的方法调用，取得集合的stream流

双列集合

双列集合无法像单列集合一样直接通过函数调用得到stream流，所以只能通过entrySet()或keySet()取得单列集合，然后再取得stream流

散元素

散元素又与以上三种方式不同，通过调用Stream类下的静态方法of(T...)来完成散元素的数据流

中间操作

filter(Predicate<T>)筛选元素

在Predicate<T>中重写test(T)方法，test(T)方法返回true时，元素保留，否则元素被过滤

map(Function<T, R>)转换流中元素的数据类型

flatMap(Function<T, R>)把流中的数据映射为新的流

通俗来讲，对stream流中的子元素做一些操作，得到一个新的stream流，新的stream流将称为原来的流中的一部分，所以当把集合中的五个元素拆分为String数组，并且把数组的stream流return后，那么现在的流中其实是拆分后的十个元素

sorted(Comparator<T>)排序

说实话这个升序和降序返回-1还是1我每次都忘

limit(int)截取元素

skip(int)跳过元素

distinct()元素去重

peek(Consumer<T>)用于观察元素的中间方法

其实和forEach(Consumer<T>)差不多，但是作为中间方法会返回原来的stream流，主要用于在开发过程中看一下元素是不是符合原来的预期

Stream.concat(Stream<T>, Stream<R>)合并两个stream

终结操作

forEach(Consumer<T>)遍历

count()统计流长度

toArray()收集流中的数据，放在数组中

collect()将流中的数据收集到集合中

将流中的数据收集到List集合中

将流中的数据收集到Set集合中

将流中的数据收集到Map集合中
